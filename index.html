3. Accessibility Service Disclosure

Critical Disclosure

Assistive Touch Pro utilizes the Android AccessibilityService API.

The Accessibility Service is used exclusively to perform physical actions requested directly by the user via the floating button menu. These actions include:

  • Performing navigation shortcuts (Back, Home, and Recents screen).
  • Triggering the device lock screen action.
  • Taking system screenshots.

Strict Privacy Commitments regarding Accessibility Service:

  • Accessibility data and operations occur entirely locally on the user's device.
  • Accessibility data is never transmitted to any external server.
  • Accessibility data is never stored, logged, analyzed, cached, or saved in any persistent or temporary storage.
  • Accessibility data is never sold, shared, or disclosed to any third party.

4. Permissions Used

To provide its core utility, Assistive Touch Pro requests the following system permissions. All permissions are used locally and solely to perform requested features:

  • Draw Over Other Apps (SYSTEM_ALERT_WINDOW): Required to display the floating touch button on top of other applications.
  • Accessibility Service (BIND_ACCESSIBILITY_SERVICE): Required to execute global system shortcuts (Back, Home, Recents, Lock Screen, Screenshot) when requested by the user.
  • Foreground Service (FOREGROUND_SERVICE & FOREGROUND_SERVICE_SPECIAL_USE): Required to keep the floating touch button active and stable in the background, preventing Android from terminating it.
  • Post Notifications (POST_NOTIFICATIONS): Required to show a persistent, low-priority silent notification, which is mandatory under Android system policy to maintain a running foreground service.
  • Receive Boot Completed (RECEIVE_BOOT_COMPLETED): Required to automatically restore the floating overlay service when the device restarts, if enabled by the user.
  • Write Settings (WRITE_SETTINGS): Required to adjust device brightness levels when the user changes them via the control panel.
  • Access Notification Policy (ACCESS_NOTIFICATION_POLICY): Required to toggle volume/do-not-disturb profiles directly from the floating panel.

5. How Information Is Used

Since we do not collect any information, we do not use, process, or analyze any user data. Any system state data (e.g., brightness levels, volume levels, flashlight status) accessed via device control shortcuts is processed on-the-fly in transient memory and is immediately discarded. No data is written to disk or sent over a network.
